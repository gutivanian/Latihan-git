Git

Git adalah sistem kpntrpl versi. Git membantu kita untuk melacak perubahan kode. Git juga berguna untuk kolaborasi dalam kode. Git dapat memanajemen proyek dengan repositori. Membuat salinan projek di local. untuk mengontrol perubahan menggunakan Staging dan Committing. Kita juga bisa bekerja di bagian versi berbeda dengan Branch dan Merge. Kita bisa mrngambil salinan dari versi terbaru dengan Pull. Kita juga bisa mengirim hasil pekerjaan kitabdengan Push.

Untuk menggunakan Git,kita bisa membuat folder atau disebut sebagai Repository. Git sekarang akan membuat sebuah folder tersembunyi untuk melacak perubahan kode. Ketika sebuah file berubah, ditambah atau dikurang, maka ini dikatakan ternodifikasi. Kita bisa memilih file yang telah dimodifikasi untuk dilakukan Staging yang berarti perubahan ke versi terbaru. File yang telah dilakukan staging dapat dilakukan Commit, yang merupakan sebuah perintah git untuk menyimpan potretan file. Git mengijinkan kita untuk melihat sejarah lengkap untuk setiap comit. Kita bisa kembali ke comit sebelumnya. Git tidak menyimpan salinan setiap comit di file terpisah, namun hanya melacak perubahan yang dilakukan pada setiap comit.

Git tidak sama dengan github. Github membuat alat yang menggunakan git. Github adalah host source code terbesar di dunia yang dimiliki microsoft sejak 2018.

Install Git

Untuk menginstall git bisa cek di www.git-scm.com. Untuk menggunakan Git, kita bisa buka git bash di windows. Kita bisa cek versi git kita dengan git --version

Konfigurasi Git

Kita bisa mengonfigurasi nama dan email dengan git config --global user.name "UserName" dan git.config --global user.email "contoh@email.com". Gunakan global untuk menset username dan email untuk semua repository di komputer kita. Jika kita ingin menset username dan email hanya untuk repo sekarang, kita bisa hapus global.

Membuat Folder Git

Kita bia membuat folder dengan mkdir NamaFolder dan berpindah ke folder itu dengan cd NamaFolder

Inisialisasi Git

Setelah kita berada pada lokasi yang benar, kita bisa menginisiasi git dengan git init. Dengan demikian kita akan membuat repository. Git akan membuat sebuah hidden folder untuk melacak perubahan file.

Menambah File Baru

Setelah membuat repo baru, repo ini adalah kosong. Jadi kita perlu menambahkan file. Misalkan kita menambahkan file HTML, maka buat file HTML dan simpan di folder yang telah dibuat. Untuk mengecek apakah sudah tersimpan, maka bisa buka kembali bash an ketikkan ls. Jika file HTMLnya muncul, maka memang telah tersimpan. Selanjutnya kita bisa melihat status dengan git status. Sekarang git mengetahui keberadaan file, namun masih belum ditambahkan ke repo. File dalam git memiiki kemungkinan berada di satu antara dua keadaan:

  1. Tracked - File yang dikenal oleh git dan telah ditambahkan ke repository
  2. Untracked - File yang berada pada direktori pekerjaan namun belum ditambahkan ke repository.

Ketika kita menambahkan file ke sebuah repository, semuanya bersifat untracked. Untuk menjadikan Tracked, kita harus melakukan Staging.
