Github

Untuk terhubung ke github, pertama-tama kita perlu membuat akun github. Selanjutnya, login github dan buat repository baru. Karena kita telah membuat repo git di lokal, kita dapat mengupload ke github atau islitahnya push. Untuk melakukannya, kita perlu copy url HTTPS dengan berbentuk "https://github.com/gutivanian/Latihan-git.git". Selanjutnya kita bisa gunakan git remote add origin https://github.com/gutivanian/Latihan-git.git. Kadang akan muncul error: remote origin already exists., artinya nama "origin" telah digunakan. Maka dari itu, kita perlu mengganti nama repo lokal kita menjadi yang lain. git remote add NamaRepo URL menspesifikasikan bahwa kita menambahkan sebuah repository remote, yang dispesifikasikan oleh "URL" sebagai "NamaRepo" ke repo git lokal. Kita dapat melakukan push master branch kita ke url

Edit Code di Github

Selain menjadi host konten Git, GitHub memiliki editor kode yang sangat bagus. Misalkan readme.md, kita bisa edit dengan mengklik tombol edit. Setelah melakukan editing, kita bisa lakukan commit perubahan. Untuk sementara kita bisa pilih "Commit directly to the master branch". Jangan lupa untuk memberi deskripsi/catatan saat melakukan commit.

Git Pull dari Github

Ketika bekerja sebagai tim di sebuah proyek, cukup penting bagi setiap orang agar selalu up to date. Kapanpun kita mulai bekerja pada sebuah proyek, kita harus mendapatkan perubahan terbaru ke salinan lokal. Dengan git, kita bisa melakukannya dengan pull. pull adalah kombinasi dari dua perintah: fetch dan merge.

Git Fetch

Fetch mendapatkan semua sejarah dari branch/repo yang terlacak. Jadi, pada local git, fetch mengupdate untuk mengetahui apa yang telah terjadi pada github. 

            git fetch origin
            remote: Enumerating objects: 5, done.
            remote: Counting objects: 100% (5/5), done.
            remote: Compressing objects: 100% (3/3), done.
            remote: Total 3 (delta 2), reused 0 (delta 0), pack-reused 0
            Unpacking objects: 100% (3/3), 733 bytes | 3.00 KiB/s, done.
            From https://github.com/w3schools-test/hello-world
               e0b6038..d29d69f  master     -> origin/master

Sekarang kita telah mendapatkan perubahan terakhir. Kita dapat mengecek statusnya: 

            git status
            On branch master
            Your branch is behind 'origin/master' by 1 commit, and can be fast-forwarded.
              (use "git pull" to update your local branch)
            
            nothing to commit, working tree clean

Kita berada pada "origin/master" oleh 1 commit. Kita bisa mengecek log commit dengan git log origin/master. Kita juga bisa melihat perbedaan antara master lokal dengan origin/master dengan git diff origin/master.

Git Merge

merge mengkombinasikan branch sekarang dengan branch lain yang dipilih. Kita bisa mengonfirmasi update dengan memerge/menggabungkan branch sekarang (master) dengan origin/master. Kita bisa melakukan melakukan merge dengan git merge origin/merge.

Git Pull

Pull adalah proses mengupdate repository lokal tanpa melewati proses panjang diatas. pull adalah gabungan antara fetch dan merge. Ini berguna untuk menarik semua perubahan dari repository remote ke branch tempat kita bekerja. Misalkan kita edit readme lagi, kita bisa gunakan pull untuk mengupdate git local kita.

Git Push

Misalkan kita membuat beberapa perubahan di git lokal dan kemudian push mereka ke github. Kita lakukan commit terlebih dahulu misalkan git commit -a -m "Message Commit". Selanjutnya kita bisa melakukan push untuk menambahkan perubahan yang kita lakukan ke git remot dengan git push origin
