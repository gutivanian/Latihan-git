Git Commit

Ketika pekerjaan telah selesai, kita dapat berpindah dari stage ke commit. Menambahkan commit akan melacak perkembangan kita dan perubahan selama kita bekerja. Git menganggap setiap commit adalah titik perubahan atau "save point". Ini adalah titik pada sebuah projek dimana kita bisa kembali jika kita menemukan bug atau ingin melakukan perubahan.

Ketika melakukan commit, kita harus memberikan pesan. Dengan menambahkan pesan yang jelas, kita akan memberikan informasi mengenai apa yang berubah dan kapan. Contoh kita bisa melakukan commit dan memberi pesan dengan git commit -m "Pesan": 

            git commit -m "First release of Hello World!"
            [master (root-commit) 221ec6e] First release of Hello World!
             3 files changed, 26 insertions(+)
             create mode 100644 README.md
             create mode 100644 bluestyle.css
             create mode 100644 index.html

Perintah commit akan melakukan commit, sedangkan perintah -m "Pesan" adalah penambahan pesan. Staging Environment telah berhasil dicommit ke repo dengan pesan "First release of Hello World!".

Git Commit Tanpa Stage

Kadang-kadang, ketika melakukan perubahan kecil, menggunakan staging environment tidaklah sesuai dan buang-buang waktu. Maka dari itu, memungkinkan untuk melakukan commit perubahan secara langsung melewati staging environment. Opsi -a akan secara otomatis melakukan staging setiap ada perubahan dari file yang telah terlacak. 

            git commit -a -m "Updated index.html with a new line"
            [master 09f4acd] Updated index.html with a new line
             1 file changed, 1 insertion(+)

Hati-hati bahwa melewati staging sangat tidak direkomendasikan. Melewati step staging kadang-kadang akan memunculkan perubahan yang tidak diinginkan.

Git Commit Log

Untuk melihat sejarah commit dari repository, kita bisa gunakan perintah git log
