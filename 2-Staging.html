Git Staging Environment

Salah satu fungsi utama dari Git adalah konsep Staging Environment dan Commit. Ketika kita bekerja, kita mungkin menambahkan, mengedit, atau menghapus file. Namun ketika kita sampai ke titik tertentu atau telah menyelesaikan pekerjaan, kita perlu menambahkan file ke Staging Environment.

File dikatakan Staged jika file telah siap untuk dicommit ke repository tempat kita bekerja. Misalkan kita telah selesai bekerja dengan file HTML, maka kita bisa menambahkan ke Staging Environmen dengan git add NamaFile.html. Kemuian kita dapat mengecek status dengan git status dan akan menghasilkan 

            git status
            On branch master
            
            No commits yet
            
            Changes to be committed:
              (use "git rm --cached ..." to unstage)
                new file: index.html

Sekarang terlihat bahwa file sudah berada di Staging Environment.

Menambah Lebih dari Satu File

Kita bisa menambahkan banyak file ke Staging Environment dengan menggunakan git add --all. Menggunakan --all ketimbang menggunakan nama file masing-masing akan melakukan staging ke semua perubahan (file baru, berubah atau hilang). Cara lain adalah dengan git add -A, hasilnya sama saja.
