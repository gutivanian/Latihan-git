Git Branch

Di git, sebuah branch(cabang) adalah versi baru/terpisah dari repository utama. Misalkan kita memiliki proyek yang sangat besar, dan misalkan kita ingin melakukan update desainnya. Apa yang perlu dilakukan dengan dan tanpa git adalah:

Tanpa git:

  • Membuat salinan dari semua file relevan untuk menghindari efek dari versi yang sedang berjalan.
  • Mulai bekerja dengan desain dan mencari tahu kode yang bergantung pada file lain, yang iaman ini juga perlu untuk diubah
  • Membuat salinan dari file yang bergantung dengan file lain juga. Kita juga perlu memastikan bahwa setiap dependensi file merefer ke nama file yang benar
  • DARURAT! Ada error yang tidak berhubungan di suatu tempat di proyek, sehingga perlu diselesaikan secepatnya
  • Simpan semua file, buat catatan nama dari salinan tempat kita bekerja
  • Bekerja dengan error yang tidak berkaitan dan update kode untuk menyelesaikannya
  • Kembali ke desain, dan selesaikan pekerjaan disini
  • Salin kode, atau ubah nama, sehingga desain yang telah berubah sekarang berada pada versi yang berjalan
  • Akan ada masalah lain yaitu bahwa jika menyalin file sebelum diperbaiki, error akan muncul lagi

Dengan git:

  • Dengan cabang(branch) yang disebut sebagai desain-baru, kita bisa edit kode tanpa memengaruhi cabang utama
  • DARURAT! Ada error yang tidak berhubungan di suatu tempat di proyek, sehingga perlu diselesaikan secepatnya
  • Buat cabang baru dari proyek utama dan kasih nama misalkan "small-error-fix"
  • Selesaikan error itu an gabungkan cabang "small-error-fix" dengan cabang utama.
  • Kembali ke cabang desain-baru, dan selesaikan pekerjaan disini
  • Gabungkan cabang desain-baru dengan cabang utama (akan diinformasikan bahwa error yang telah diselesaikan telah menghilang)

Cabang/Branch mengijinkan kita untuk bekerja pada bagian berbeda dari sebuah proyek tanpa memengaruhi cabang utama. Ketika pekerjaan selesai, sebuah cabang dapat digabungkan dengan cabang utama. Kita bahkan dapat berpindah antar cabang dan bekerja pada proyek berbeda tanpa memengaruhi satu sama lain. Membuat cabang di git sangat mudah dan cepat.

Membuat Cabang Git Baru

Misalkan buat sebuah fitur pada file lama. Kita bekerja di repository lokal, dan tidak ingin mengganggu atau bahkan merusah cabang utama. Sehingga kita bisa membuat cabang baru. Untuk membuatnya gunakan git branch nama-branch 

            git branch hello-world-images

Kita telah membuat sebuah branch baru. Untuk mengecek apakah branch tersebut benar-benar telah terbuat, kita bisa cek melalui git branch 

            git branch
              hello-world-images
            * master

Kita bisa melihat terdapat branch baru bernama "hello-world-images" dan sekarang kita sedang bekerja pada branch master. Kita bisa melihat bahwa tanda bintang (*) berada di branch master. Kita bisa berpindah cabang dengan perintah checkout 

            git checkout hello-world-images
            Switched to branch 'hello-world-images'

Sekarang kita telah berpindah dari cabang master ke cabang baru. Kita bisa melakukan perubahan dalam file, misalkan file HTML ini. Maka kita dapat melihat status cabang sekarang dengan git status. 

            $ git status
            On branch branch-1
            Changes not staged for commit:
              (use "git add <file>..." to update what will be committed)
              (use "git restore <file>..." to discard changes in working directory)
                    modified:   5-Git-Branch.html
            
            no changes added to commit (use "git add" and/or "git commit -a")

Terlihat bahwa perubahan yang kita buat masih belum berada pada staging environment, Sehingga kita perlu menambahkan dengan git add --all. Kemudian kita bisa lakukan commit ke branch dengan git commit -m "Pesan commit". Dengan demikian, kita telah mendapatkan sebuah branch baru yang terpisah dari branch master.

Berpindah Antar Branch

Sekarang kita ingin mencoba untuk berpindah cabang. Sekarang kita berada pada cabang "branch-1" atau "hello-world-images". Kita bisa mengecek fila apa saja pada cabang ini dengan "ls". Dan kemudian kita berpindah ke cabang master dengan git checkout master dan kemudian "ls", kita bisa melihat bahwa perubahan dan penambahan file tidak berpengaruh ke branch master, karena proses itu terjadi pada branch-1.

Branch Darurat

Namun, misalkan kita belum selesai bekerja di cabang "branch-1" dan memiliki kebutuhan untuk menyelesaikan error di master. Kita tidak ingin berurusan dengan master secara langsung dan juga tidak ingin merusah branch-1 karena masih belum selesai. Sehingga kita bisa membuat branch baru untuk mengerjakan hal darurat dengan git checkout -b emergency-fix. 

            it checkout -b emergency-fix
            Switched to a new branch 'emergency-fix'

Setelah berpindah ke branch emergency-fix, kita dapat menyelesaikan kode kita untuk error dan kemudian ditambahkan ke staging environment dan juga dilakukan commit. Maka selesesai sudah penyelesaian untuk master dan siap untuk menggabungkan dua branch.
